Scribus on OS/2 and eComStation

Thanks to Paul Smedley, the Scribus Team is happy to provide a port of Scribus to OS/2 and its successor eComStation.

Downloads

Scribus 1.3.3.x Stable

The latest port of Scribus 1.3.3x is available from Sourceforge. Please select the ZIP archive with "os2" in the file name for download. The archive contains a folder called "scribus," in which you find an installer called "scribus.exe".

Scribus 1.3.6+ Development

You can download a ZIP archive containing an installer for the development version either from

Sourceforge or Paul Smedley's Unix Ports for OS/2 site.

Other requirements

To run Scribus you need at least OS/2 Warp 4 or eComStation. Scribus will not run on older versions of OS/2.

In addition, please note that you need to install the following software:

  • A copy of libc063.dll, which can currently be downloaded from ftp://ftp.netlabs.org/pub/gcc/libc-0.6.3-csd3.zip and gcc434.dll, which can be downloaded from http://download.smedley.info/gcc434.zip. Both files need to copied to C:\OS2\DLL (OS/2) or C:\ECS\DLL (eComStation).
  • Ghostscript and Python 2.6 (available from http://os2ports.smedley.info) are required to support some functionality. For instance, Python scripts won't work if Python is missing. The print preview and EPS import will not work if Ghostscript isn't available. If Scribus doesn't detect Ghostscript automatically, you can set the path to the GS exectutable (called "gsos2.exe") in the Scribus Preferences under "External Tools."
  • Scribus 1.3.6 or later also requires the Qt4 GA dll's that can be downloaded as a WarpIn package from http://svn.netlabs.org/qt4
    • .

Known issues

  • Printing support to "normal" OS/2 printers is not complete, but PDF export is. You can easily print via Acrobat Reader or another PDF viewer as a temporary workaround.
  • Printing to CUPS printers is supported. File > Print will detect all available CUPS printers.
